GYMPRO DASHBOARD VALIDATION + ERROR HANDLING REFERENCE
======================================================

Scope
-----
This file maps sidebar pages in each dashboard to:
1) Frontend input validation and user-visible error/success messages.
2) Backend route guards, schema validation, and error handling.
All references include file + line numbers.

Global Backend Guards/Error Formatter
-------------------------------------
- backend/src/shared/middleware/auth/requireOwnership.js:6,13,37
  Ownership checks; returns 401/403 messages like 'Unauthorized' and 'Forbidden: ownership mismatch'.
- backend/src/shared/middleware/errorHandler.js:6,17
  Normalizes server errors into JSON response with message and status.

Notes
-----
- Some sidebar pages are read-only dashboards; they have API load catch/fallback handling but no form validation.
- Ownership and role checks are enforced primarily in backend routes via authenticateJWT, authorizeRoles, and requireOwnership.
- Validation failures are generally returned as 'Validation failed' with field details from safeParse/parse wrappers.
